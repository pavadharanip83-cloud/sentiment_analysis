About the Project

Twitter Sentiment Analysis using Support Vector Machines.

📌 Project Overview

SentimentAI is a machine learning system that classifies Twitter tweets into three sentiment categories: Positive, Negative, and Neutral. It uses a Support Vector Machine (SVM) with TF-IDF vectorization — the best-performing traditional ML approach for short, noisy social media text.

🎯 Objectives

1. Build a robust tweet preprocessing pipeline using NLTK.

2. Train and evaluate multiple ML models to identify the best performer.

3. Deploy a real-time REST API and professional web interface.

4. Compare SVM against Naive Bayes, LR, RF and XGBoost.

🧠 Why SVM?

Support Vector Machines find the optimal decision boundary (hyperplane) that maximises the margin between sentiment classes in high-dimensional TF-IDF space. Unlike tree-based models, SVMs excel on sparse, high-dimensional text data — making LinearSVC the gold standard for traditional NLP classification tasks.

🔧 Preprocessing Pipeline

Each tweet undergoes: lowercasing → URL removal → @mention removal → hashtag cleaning → RT stripping → punctuation removal → stopword filtering (preserving negations like "not", "never") → lemmatization using WordNetLemmatizer. This produces clean, normalised tokens ready for TF-IDF encoding.

⚙️ Technology Stack

  • Python Core language for ML pipeline and backend
  • scikit-learn LinearSVC, TF-IDF vectorizer, model evaluation
  • NLTK Tokenization, stopword removal, lemmatization
  • Flask REST API backend with CORS support
  • HTML/CSS Custom professional frontend (no frameworks)
  • Chart.js Interactive performance charts on dashboard
  • Kaggle Twitter dataset for training and evaluation
